# 📸 가족 추억 보관 상자 v28

### ✨ 주요 기능

- 🔒 보안 100% — 사진이 클라우드에 올라가지 않습니다. 집 Wi-Fi 안에서만 이동하므로 해킹 위험 제로
- 🖼️ 원본 무손실 전송 — 화질 압축 없이 원본 그대로 전송 (SHA-256 무결성 검증)
- 📁 자동 폴더 정리 — 외장하드에 날짜/장소명 폴더 자동 생성 (예: `2024-03-15/하남스타필드`)
- 📱 핸드폰 용량 절약 — 전송 후 원본 삭제 가능, 앱에서 99% 용량 줄인 미리보기로 확인 가능
- 🗺️ GPS 위치 자동 변환 — 사진 속 GPS 좌표를 실제 장소 이름으로 자동 변환 (Nominatim/OpenStreetMap, 무료)
- 💾 외장하드 자동 감지 — USB 외장하드 꽂으면 자동으로 사진 이관
- 🌐 웹 전송 지원 — 앱 설치 없이 핸드폰 브라우저에서도 사진 전송 가능
- 🗑️ 백업 갤러리 선택/삭제 — 날짜별 전체 선택, 개별 체크, 서버 꺼져있어도 삭제 가능
- 📴 오프라인 갤러리 — 서버(노트북) 꺼져있어도 썸네일/동영상 미리보기 즉시 확인 가능
- 🎬 동영상 미리보기 — 전송된 동영상을 360p로 변환하여 핸드폰에 저장, 서버 없이 재생
- 🎥 동영상 자동재생 + 터치잠금 — 백업된 동영상만 필터하여 연속/반복 재생, 재생 중 터치 잠금
- 📱 백그라운드 전송 — 다른 앱 사용 중에도 전송 계속, 알림바에서 진행률 확인
- 📋 전송 실패/중복 사유 표시 — 왜 실패했는지, 언제 중복 전송했는지 상세 표시
- ☁️ **중복 사진 자동 표시** — 전송 시 중복 판별된 사진에 구름 아이콘 표시, 재전송 방지 (v28 신규)
- 🔧 **빌드 스크립트 안정화** — 한글 경로 호환성 개선, 각 단계 검증 추가 (v28 신규)
- 💰 100% 무료 — 클라우드 비용, 월정액 없음

### 🆕 v28 업데이트 내용

#### ☁️ 중복 사진 자동 표시
사진 전송 시 서버가 **"이미 전송된 파일"**이라고 응답하면, 해당 사진에 ☁️ 구름 아이콘이 자동으로 표시됩니다.

**이전 (v27):**
- 중복으로 판별된 사진은 전송 결과에만 표시
- 갤러리에서는 중복인지 알 수 없어서 다음에 또 선택할 수 있었음

**이후 (v28):**
- 중복 판별된 사진에도 ☁️ 구름 아이콘이 즉시 표시됨
- 다음 전송 시 이미 서버에 있는 사진을 한눈에 확인 가능
- "전송됨만" 필터로 중복 포함 전체 전송 완료 사진 조회 가능

> 💡 중복 = 이미 서버에 존재하는 파일이므로, 전송 완료와 동일하게 구름 아이콘으로 표시합니다.

#### 🔧 빌드 스크립트 안정화
APK 빌드 스크립트(`build_apk.ps1`)의 안정성을 개선했습니다.

- `Get-Content -Raw` → `Get-Content | Out-String`으로 PowerShell 호환성 확보
- 매 단계(버전 업데이트, 소스 동기화, APK 생성, 복사)마다 검증 로직 추가
- 검증 실패 시 즉시 중단 (`exit 1`)
- `cmd /c copy`로 한글 경로 파일 복사 안정화

### 📱 앱 vs 🌐 웹 — 사진을 보내는 2가지 방법

| | 📱 앱 (APK 설치) | 🌐 웹 (브라우저) |
|--|---|---|
| **설치** | APK 설치 필요 | ❌ 설치 불필요 |
| **사진 전송** | ✅ | ✅ |
| **갤러리에서 바로 선택** | ✅ 편하게 선택 | ❌ 파일 탐색기에서 선택 |
| **전송된 사진 표시** | ✅ 초록 체크 표시 | ❌ 표시 없음 |
| **전송 후 원본 삭제** | ✅ 앱에서 바로 삭제 | ❌ 직접 삭제 |
| **썸네일 미리보기** | ✅ 백업 갤러리 | ❌ 없음 |
| **전송 진행률** | ✅ 실시간 | ✅ 실시간 |
| **백그라운드 전송** | ✅ 다른 앱 사용 가능 | ❌ 브라우저 유지 필요 |
| **추천 대상** | 본인 (주로 쓰는 사람) | 가족·친구 (가끔 보내는 사람) |

> 💡 본인은 **앱**으로 쓰고, 앱 설치가 귀찮은 가족에게는 **웹 주소만 알려주면** 됩니다!

> ⚠️ **웹 주소는 왜 링크가 아닌가요?**
> 
> `http://192.168.x.x:8000/web/` 에서 x.x 숫자는 **집마다 다릅니다.**
> 이건 인터넷 주소(naver.com)가 아니라, **집 Wi-Fi 안에서만 존재하는 내부 주소**예요.
> 서버를 실행하면 파란 창에 본인의 주소가 표시됩니다. 그 주소를 가족에게 알려주세요.
> 집 Wi-Fi가 바뀌지 않는 한 주소는 항상 같으니, 한 번만 알려주면 됩니다.

#### 📱 핸드폰 용량 절약 예시

사진을 서버로 전송한 뒤 핸드폰에서 원본을 삭제해도, **앱에서 아주 작은 미리보기(썸네일)**로 사진을 계속 볼 수 있습니다.

| 사진 수 | 원본 용량 | 앱에 남는 썸네일 용량 | 절약되는 용량 |
|--------|----------|-------------------|------------|
| 1,000장 | 약 5GB | 약 30MB | **4.97GB 절약** (99.4%) |
| 10,000장 | 약 50GB | 약 300MB | **49.7GB 절약** |
| 100,000장 | 약 500GB | 약 3GB | **497GB 절약** |

> 예를 들어 핸드폰에 사진 1만 장(50GB)이 있다면, 전송 후 원본을 지우면 **49.7GB가 비워집니다.**
> 앱에는 300MB짜리 작은 미리보기만 남아서, 언제든 사진을 훑어볼 수 있어요.

### 📥 설치 방법 (처음부터 끝까지)

이 앱은 **핸드폰 앱 + 노트북 서버** 두 가지를 모두 설치해야 동작합니다.

> 💡 **왜 서버도 설치해야 하나요?**
> 
> 이 앱은 구글 포토처럼 인터넷 서버를 쓰지 않고, **내 노트북이 서버 역할**을 합니다.
> 핸드폰에서 사진을 보내면 노트북이 받아서 정리해주는 구조라서,
> 사진을 받아줄 노트북 프로그램(서버)이 필요한 거예요.
> USB 케이블로 파일을 옮길 때 PC가 켜져 있어야 하는 것과 같은 원리입니다.

#### 🔹 1단계: 노트북에 Python 설치하기

Python은 서버를 실행하기 위한 프로그래밍 도구입니다. (무료)

1. 브라우저에서 **https://www.python.org/downloads/** 접속
2. 노란색 **"Download Python 3.xx.x"** 버튼 클릭하여 다운로드
3. 다운로드된 설치 파일 실행
4. ⚠️ **중요!** 설치 화면 맨 아래에 **"Add Python to PATH"** 체크박스가 있습니다. **반드시 체크**하세요!
5. **"Install Now"** 클릭 → 설치 완료까지 기다리기

**설치 확인 방법:**
- 키보드에서 `Windows키 + R` 누르기
- `cmd` 입력 후 엔터
- 검은 창에 `python --version` 입력 후 엔터
- `Python 3.xx.x`가 나오면 성공! ✅

---

#### 🔹 1-2단계: 노트북에 Git 설치하기

Git은 서버 코드를 다운로드하고 **자동 업데이트**하기 위한 도구입니다. (무료)

1. 브라우저에서 **https://git-scm.com/download/win** 접속
2. **"Click here to download"** 클릭하여 다운로드
3. 다운로드된 설치 파일 실행
4. 설치 옵션은 **모두 기본값 그대로** "Next" → "Next" → ... → "Install" 클릭
5. 설치 완료!

**설치 확인 방법:**
- 키보드에서 `Windows키 + R` 누르기
- `cmd` 입력 후 엔터
- 검은 창에 `git --version` 입력 후 엔터
- `git version 2.xx.x`가 나오면 성공! ✅

---

#### 🔹 2단계: 서버 코드 다운로드하기

1. 키보드에서 `Windows키 + R` 누르기
2. `cmd` 입력 후 엔터
3. 검은 창에 아래 명령어를 **복사해서 붙여넣고** 엔터:

```
git clone https://github.com/abraxass0511-lab/family-photo-archive.git "%USERPROFILE%\Desktop\family-photo-archive"
```

4. 바탕화면에 `family-photo-archive` 폴더가 생깁니다

> 💡 이 방법으로 설치하면, 나중에 서버 업데이트가 있을 때 **명령어 한 줄**로 최신 버전을 받을 수 있습니다! (아래 "🔄 서버 업데이트 방법" 참고)

---

#### 🔹 3단계: 서버 준비하기 (최초 1회만 하면 됩니다)

1. 위에서 받은 폴더 안의 **`server`** 폴더를 열기
2. 폴더 빈 공간에서 **Shift + 마우스 우클릭** → **"여기서 PowerShell 창 열기"** 클릭
3. 파란 창이 열리면, 아래 명령어를 **한 줄씩** 복사해서 붙여넣고 엔터:

**첫 번째 줄** (가상환경 만들기):
```
python -m venv venv
```

**두 번째 줄** (가상환경 켜기):
```
.\venv\Scripts\activate
```

**세 번째 줄** (필요한 프로그램 설치):
```
pip install -r requirements.txt
```

> ⏳ 세 번째 명령어는 필요한 프로그램들을 자동으로 다운로드합니다.
> 인터넷 속도에 따라 **3~10분** 걸릴 수 있어요.
> 화면에 글자가 계속 나오고 있으면 정상입니다. 기다리면 됩니다!

---

#### 🔹 4단계: 서버 실행하기

1. `family-photo-archive` 폴더에 있는 **`start_server.ps1`** 파일을 **우클릭**
2. **"PowerShell에서 실행"** 클릭
3. 파란 창이 열리면서 아래와 비슷한 내용이 표시됩니다:
```
  Phone Access URL:
  http://192.168.0.5:8000/web/
```
4. 여기 나오는 **`http://192.168.0.5:8000`** ← 이 주소를 메모하세요!
   (숫자는 사용자마다 다릅니다)

> 💡 이 파란 창을 닫으면 서버가 꺼집니다.
> **사진을 전송하는 동안에는 이 창을 열어두세요.** 전송이 끝나면 닫아도 됩니다.

---

#### 🔹 5단계: 핸드폰에 앱 설치하기

1. 핸드폰 브라우저에서 이 페이지에 접속합니다
2. 아래쪽 **Assets** 섹션에서 **`포토백업_v28.apk`** 를 터치하여 다운로드
3. 다운로드 완료 후 알림을 터치해서 설치 시작
4. **"출처를 알 수 없는 앱"** 경고가 뜨면:
   - "설정" 터치 → "이 출처 허용" 켜기 → 뒤로 → 다시 설치
5. 설치 완료 → **"열기"** 터치

---

#### 🔹 6단계: 앱에서 서버 연결하기

1. 앱을 열면 **설정** 탭이 보입니다
2. **서버 주소** 칸에 4단계에서 메모한 주소를 입력합니다
   - 예시: `http://192.168.0.5:8000`
3. **"연결 테스트"** 버튼을 눌러서 ✅ 연결 성공이 나오는지 확인
4. 연결 성공이면 **전송 탭**으로 이동 → 갤러리에서 사진 선택 → 전송!

> ⚠️ **핸드폰과 노트북이 같은 Wi-Fi에 연결되어 있어야 합니다!**
> 집 와이파이에 둘 다 연결하세요.

### 🔄 서버 업데이트 방법

서버에 새로운 기능이 추가되거나 버그가 수정되면, **명령어 한 줄**로 최신 버전을 받을 수 있습니다.
앱(APK)은 다시 설치할 필요 없이, 서버만 업데이트하면 됩니다.

1. `family-photo-archive` 폴더의 **`server`** 폴더를 열기
2. 폴더 빈 공간에서 **Shift + 마우스 우클릭** → **"여기서 PowerShell 창 열기"** 클릭
3. 아래 명령어를 복사해서 붙여넣고 엔터:

```
git pull origin master
```

4. 끝! 서버를 다시 실행하면 최신 버전이 적용됩니다.

> 💡 서버 업데이트는 **앱(APK) 재설치 없이** 적용됩니다.
> 앱은 그대로 두고 서버만 업데이트하세요.

### ❓ 자주 묻는 질문

**Q. 노트북 꺼져 있어도 전송 돼?**
→ 아닙니다. 노트북이 켜져 있고 서버(파란 창)가 실행 중일 때만 전송 가능합니다.

**Q. 노트북 꺼져 있어도 사진을 볼 수 있어?**
→ 네! 동기화 시 썸네일과 동영상 미리보기가 핸드폰에 저장됩니다. 서버 없이도 갤러리를 바로 볼 수 있어요.

**Q. 전송 중에 다른 앱을 쓰면?**
→ 백그라운드 전송을 지원합니다! 카카오톡, 유튜브 등 다른 앱을 사용해도 전송이 계속됩니다. 상태바를 내리면 진행률을 확인할 수 있어요.

**Q. 사진 화질이 떨어져?**
→ 아닙니다. 원본 그대로 1바이트도 변하지 않고 전송됩니다. 핸드폰에서 보는 것은 저화질 미리보기이고, 원본은 외장하드에 보관됩니다.

**Q. 해킹 위험은?**
→ 사진이 인터넷(클라우드)을 거치지 않고 집 Wi-Fi 안에서만 이동하므로 보안 100%입니다.

**Q. 핸드폰에서 원본 지우면 다시 못 봐?**
→ 앱에서 99% 용량 줄인 썸네일(미리보기)로 계속 볼 수 있어요. 원본은 외장하드에 있습니다.

**Q. 서버 꺼져있을 때 삭제하면 어떻게 돼?**
→ 핸드폰에서 즉시 삭제되고, 서버에는 다음 동기화 때 자동 반영됩니다.

**Q. 전송이 중복이라고 뜨는데?**
→ 이미 서버에 전송된 파일입니다. v28부터는 중복 판별된 사진에 ☁️ 구름 아이콘이 자동 표시되어, 다음에 같은 사진을 또 보내는 것을 방지합니다.
